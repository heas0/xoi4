# Развертывание проекта на GitHub Pages

Так как в вашем проекте уже установлен пакет `gh-pages` и настроены скрипты (`predeploy` и `deploy` в `package.json`), а также правильно указан `base: '/xoi4/'` в `vite.config.ts`, процесс развертывания максимально упрощен. 

Выполните следующие шаги в терминале:

## 1. Инициализация и сохранение кода (если Git был удален)

Если вы еще не инициализировали Git заново:

```bash
git init
git add .
git commit -m "Initial commit for GitHub Pages deployment"
```

## 2. Связка с репозиторием GitHub

Создайте пустой репозиторий на GitHub с названием **`xoi4`** (это важно, так как путь в `vite.config.ts` настроен именно на эту папку). Не создавайте файлы README или .gitignore через интерфейс GitHub, чтобы репозиторий был полностью пустым.

Затем свяжите локальный проект с GitHub и отправьте исходный код:

```bash
git remote add origin https://github.com/heas0/xoi4.git
git branch -M main
git push -u origin main
```
*(Обязательно замените `heas0/xoi4.git` на ваш актуальный логин и название репозитория, если они отличаются)*

## 3. Деплой (развертывание)

Запустите скрипт деплоя. Он автоматически соберет проект в папку `dist` и отправит только её в специальную ветку `gh-pages`:

```bash
npm run deploy
```

## 4. Включение GitHub Pages в настройках

После того как скрипт успешно завершит работу:
1. Зайдите на страницу вашего репозитория на GitHub.
2. Перейдите во вкладку **Settings** (Настройки).
3. В левом меню выберите пункт **Pages**.
4. В разделе *Build and deployment* убедитесь, что в качестве *Source* выбрано **Deploy from a branch**.
5. Чуть ниже в пункте *Branch* выберите из выпадающего списка ветку **`gh-pages`** (папку оставьте `/(root)`).
6. Нажмите кнопку **Save**.

⏳ Подождите пару минут, пока GitHub соберет окружение. Ваш проект будет доступен по ссылке, которая появится в верхней части раздела Pages (например: `https://heas0.github.io/xoi4/`).
